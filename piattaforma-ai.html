Policy Engine Approval Workflow RAG + Explainability

AI nativa: veloce, utile, sotto controllo.

L'AI in MatevIA non e un plugin esterno: e parte del flusso operativo. Suggerisce, sintetizza, recupera contesto dai documenti, ma mantiene controlli espliciti dove serve approvazione umana.

Come agisce concretamente

1) Comprende il contesto operativo

La richiesta viene letta nel contesto del modulo attivo (CRM, Magazzino, Contabilita) per restituire output operativo, non risposte generiche.

2) Applica policy e guardrail

Le azioni sono filtrate da regole configurabili: blocco azioni vietate, approvazioni multiple su operazioni sensibili, audit delle decisioni.

3) Restituisce output spiegabile

Risposta, confidenza e note vengono presentate all'utente per supportare una decisione rapida e verificabile.

Estratto tecnico (parziale)

Di seguito un piccolo estratto dal motore policy AI (senza pubblicare il codice completo):

if (executeWithConfirmation)
{
    var doubleApproval = cfg.RequireDoubleApprovalActions ?? Array.Empty<string>();
    if (doubleApproval.Any(a => string.Equals(a?.Trim(), key, StringComparison.OrdinalIgnoreCase)))
    {
        return new AiPolicyDecision(
            Allowed: false,
            ErrorCode: "AI_APPROVAL_REQUIRED",
            RequiresApproval: true,
            RequiredApprovals: Math.Max(2, cfg.DefaultRequiredApprovals));
    }
}

Inoltre la configurazione AI prevede redazione dati sensibili attiva di default: EnableSensitiveDataRedaction = true.
